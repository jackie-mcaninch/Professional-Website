Projects

  • Libdogecoin

    This is a link to the Dogecoin Foundation's open-source "Libdogecoin" project, which essentially simplifies the core functionality of Dogecoin Core into one clean, portable C library that is easy for new developers to understand. Staying in close communication with Core developers, I made some significant contributions to this repository, including the high-level Python and Golang wrappers for address and transaction functionality. Additionally, I created several educational docs describing the API and other insights into cryptocurrency functions, which can be found in the doc directory. I plan to continue working with them in the second phase of development for this library.

  • Green Lightning Coin (GIGI)

    This repo is the first rough draft of our novel implementation of "Proof-of-Space" consensus. Cryptocurrency's current "Proof-of-Work" consensus algorithm require massive amounts of power to continuously generate hashes in order to mine blocks; with proof-of-space, these hashes are generated only once and reused throughout the miner's lifetime, saving a countless amount of energy. This algorithm concept allows us to build GIGI: a blockchain-based cryptocurrency network that is greener, faster, and more suitable to small transactions than leading crypto names. My contribution-- the "Vault"-- is the collection of functions required to validate transactions using Proof-of-Space methods, modeled after a much simpler implementation of Chia. Current state of the project resides here, but contains intellectual property of Illinois Tech. Please contact me for access to the full repository.

  • Buy Nothing Web RESTful API

    This project is to simulate the functionality of a "Buy Nothing" system, where users use the API to communicate and share items or services based on geographical location. This is a bare bones Java project which only employs simple HTTP requests inside a REST API. In the end, this project did help me learn about how to host a local web server using Apache Tomcat, as well as sending and receiving class information by JSON transport.

  • Zoom Chat Monitoring App

    This app is intended for teachers who use Zoom for their online teaching purposes. There are a few functions it can perform: checking for chat participation, flagging inappropriate language, or searching for chats from a specific student. During this time of increased remote classwork, this can be a useful tool for teachers to obtain quick stats on their students' involvement in the Zoom classroom, through smooth interaction on a user-friendly GUI.

  • Online Grocery Store Database

    This project uses Java and MySQL to emulate the UI for a online grocery ordering service. Included is both a customer view and employee view, each allowing the user to edit his/her information and perform actions, such as buy items or stock items. This sort of application is useful for those who may not have the ability to go to actual stores, whether that be for health concerns or general accessibility. This project aims to support online purchases by keeping track of user information and inventory of stock.

  • NP-Hard Optimization: Separating Points by Axis-Parallel Lines

    This project takes a set of points as input and outputs a minimal set of horizontal and vertical lines that isolate every point in the input set. This was implemented in two ways: local optimization, where a non-minimal but feasible solution is reduced to a minimal solution, and greedy, where the solution is built from a starting point of zero lines and adds as necessary.

  • Google Hash Code Challenge Submission

    This smaller project was my submission for the 2021 Google Hash Code machine learning competition. The problem to solve was to control a system of stop lights to maximize the amount of cars who could finish their given path within a certain time frame.
